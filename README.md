# TeamFlow Lite

一个供内部小团队使用的轻量协作产品，独立于仓库中的其他产品，可单独运行和部署。

## 已包含的功能

- 团队成员：添加成员、停用状态、四级角色权限（负责人 / 管理员 / 成员 / 只读成员）
- 产品需求：提出需求、优先级、负责人、目标日期、评审及研发状态流转
- 需求拆解：一键生成澄清、方案、开发、验收四阶段基础任务，再由团队手动调整
- 团队任务：任务看板、负责人、优先级、截止时间、状态更新
- 时间节点：为需求设置评审、验收、上线等里程碑
- 提醒追踪：根据团队设置提前提示即将到期和已经逾期的任务、节点
- 团队动态：记录关键创建和更新动作
- 数据持久化：无外部数据库依赖，数据保存在 JSON 文件中

## 本地运行

需要 Node.js 18 或更高版本。

```bash
cd teamflow-lite
npm start
```

打开 <http://localhost:7360>。

首次运行会自动创建演示数据和三个账号：

| 角色 | 邮箱 | 本地初始密码 |
| --- | --- | --- |
| 负责人 | `admin@team.local` | `teamflow123` |
| 管理员 | `product@team.local` | `product123` |
| 成员 | `dev@team.local` | `dev12345` |

生产环境应设置 `INITIAL_ADMIN_PASSWORD`，并在第一次运行前保持数据目录为空。数据默认写入 `teamflow-lite/data/teamflow.json`，可用 `DATA_DIR` 修改目录。

## 独立部署

`render.yaml` 已包含 Render Web Service 和持久磁盘配置。在 Render 中将 Root Directory 设置为 `teamflow-lite`，使用该目录内的 Blueprint 即可。也可以用任意 Node 托管平台：

- 启动命令：`npm start`
- 健康检查：`/api/health`
- 服务端口：读取平台的 `PORT` 环境变量
- 持久目录：设置 `DATA_DIR` 到持久磁盘
- 首次管理员密码：设置 `INITIAL_ADMIN_PASSWORD`

## 检查与测试

```bash
npm run check
npm test
```

冒烟测试会在临时目录启动独立服务，验证登录、创建需求、自动拆解、完成任务和工作台统计，不会改动正式数据。

## 当前定位

这是内部使用的轻量 MVP。提醒目前是产品内提醒，登录或刷新时根据任务/节点日期实时计算。后续如果需要主动推送，可在提醒数据之上增加飞书机器人、邮件或企业微信定时推送；需求拆解也预留为独立接口，后续可以把当前规则模板替换成大模型输出。
